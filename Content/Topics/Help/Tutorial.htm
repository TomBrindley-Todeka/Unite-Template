Unite Flare Threat Template Tutorial

This tutorial takes you through the process of creating a Threat Paper from the Unite Threat template.

We will cover:

  • Creating a new project from the template.

  • Setting the global variables

  • Binding the project to Source Control

  • Editing topics

  • Building (compiling) the project

  • Outputting the project to PDF file

  • We will also touch on the tools used to manage Source Control.

This is not a Flare training course, but I will answer any questions and we will touch on many of Flares tools.

If you have access to a PC running Flare you can replicate much of what we will cover in real time.

This tutorial will be available as a PDF.

Process Summary

We have created a Flare template (a dummy project) from which other Threat papers can be written. Supporting the Flare template are the styles and page layouts required for a PDF output. The assumption is that a single paper will be written by a team of authors who will review and edit each other's work, so the use of a source control system is essential.

The overall process is:

Create a new project

Set the global variables such as Threat paper name, creation date, version control

Download the latest global style sheets (Unite style and Table styles)

Commit to source control (GIT)

Close the project

Assign topics to authors

Each author creates a project from the source control master

Open a topic to edit

Edit each topic with text, tables, charts and diagrams

Commit to source control (GIT)

Build (compile) the Threat paper

Publish Threat paper

Global Project

The threat template is held in Github as a Global master (parent) project. Threat papers are linked back to this project such that certain files, such as style sheets and page layouts are always downloaed from the master prior to building an output.

Create a new project

There are three ways to create a new project:

  • Create from Template

  • Create from a previous Threat paper

  • Create from source control.

Create from Template

Select File>New Project>New Project

The New Project Wizard opens

Select language

By default, Flare creates new projects in US English. (We can change this in the Flare master files, but it will revert after every update.)

Select UK English and click Next.

The New Project source page opens and you can choose either to create from a template or create from an existing project.

  • If this is a new Threat Paper for a new company, choose Create from Template.

  • If this a revision or update to a previous company, choose Create from Existing.

Create from Template

  1. Choose the template to use (defaults to the last used template).

  2. Click next.

  3. Select Target output (normally PDF).

  4. Click Finish.

Create from Existing

  1. Click on New From Existing

  2. Click and browse to the source project.

  3. Click Next

  4. Choose the target output (normally PDF).

Create from Source control

  1. Click on Filer>New Project>Import From Source Control

  2. Click and browse to the source project.

  3. Click Next

  4. Choose the target output (normally PDF).

Bind to source control

The exact details of this are yet to be determined by your IT department. The following is based on a local GIT repository.

Note: This can be done from the new project page.

  1. Open the source control dialog: Project>Project Properties>Source Control

  2. Click Bind Project.

  3. Select the Source Control Provider (for example GIT).

  4. Type your name and email address and choose Save Per: User

  5. Click OK.

  6. Flare sets up the source control files in the local repository. From now on, you can save and COMMIT changes and these will be recorded locally and you can PUSH changes to a central repository. You can then roll back to any previous saved state.
    Flare also supports branching where you can have two separate parallel topic versions (which can be merged at a later date.

Setting the Global Variables

The Threat Template includes place holders for global variables such as:

  • Author (I suggest you set this to document lead or project manager)

  • Document Title

  • Document SubTitle

  • Document Date

  • Version (I suggest you set to 0.01, revision control is outside the scope of Flare)

These are held in the Global variableSet (Project Organiser<Variables).

Topics

The Threat paper is divided into four sections:

  • Front Matter

  • Decision Makers

  • Drivers

  • Back matter

The Front Matter includes three topics: Front Page and Contents (automatically populated), and Revisions (Manually populated).

The Decision Makers section comprises 13 topics numbered (for convenience only) "1 - xxx" to "13 -xxx".

The Drivers section comprises 21 topics numbered (for convenience only) "51 - xxx" to "71 -xxx".

The Back Matter includes three topics, (all automatically populated): Figures, Tables, Endnotes.

There are two types of content topics:

  • Leave as question

  • Change to Answer

These are easily distinguished as there is an author hint (which is not printed) stating which is which.

You can see that the (Leave as question) text is contained in text markers with a blue square. This is marking Conditional Text. The Target is set to exclude this text. You can change the text (for example, change "organisation" to "company".

<span MadCap:conditions="Default.AuthorHint">(Leave as question)</span></h1>

Endnotes

The new text includes 4 cross references: Australia.1, 118 ports,2 separate port operation.3 and subsidiaries:4

Topic (2 -What kind of organisation is it and is it part of a group?...) includes some guidance on inserting external cross references for which we use the Flare Endnotes function.

To add external references, for example to web searches, use an endnote (a type of footnote)My new endnote.

Position the cursor where you want the reference number to appear.

Select Insert/Footnote from the ribbon.

Select Endnotes Proxy.

Click OK.

The footnote appears at the bottom of the page (it will not be present on printed output).

Overtype "My new footnote" with the cross reference text.

Highlight the text an select Insert/Hyperlink.

Type the URL of the page to be referenced.

The end note will appear in a table at the end of the document (achieved using an Endnote proxy, contained in the Endnote topic.)

You can now continue working through the template topics, adding answers to the questions and including Figures and Tables.

Be aware that other authors might be working on topics. The source control function will take care of these changes and highlight any conflicts (where a topic has been changed by two authors).

You will not see the other authors' changes in real time, but you can always re sync your project.

SourceControl>Synchronise

Figures

Figures (illustrations or images) are marked by the FigureTitle paragraph tag.

The Figures will be consecutively numbered at build time

The list of figures will appear in the Figures topic at build time.

Tables

Tables are created using the Insert>Table function. Flare table styles are defined by their own CSS and stored in the Resources>TableStyles folder. The layout and appearance are determined by the table style.

Examples of tables are contained within the template (but not in the output) in topic: Table Examples.

The simplest way to create a table is using the table snippets.

To create a table:

Type a table title and tag it p.TableTitle. This will create an entry in the Table of Tables.

Styles

The cascading style sheet Unite Master Style Sheet.css contains all the styles you need to create a threat template.

Examples of styles are shown in the topic Styles

If you need to create a style, request this from the Flare administrator. The master style sheet will be reapplied each time the Threat paper is built, so creating a local style will be overwritten.

Table of Contents

There are three TOCs:

  • A parent TOC (2022-07-25 Unite Threat Sample (Master)

  • A subsidiary TOC (TOC-Decisions Makers)

  • A subsidiary TOC (TOC-Drivers Makers)

The TOCs determine which topics are included in the output.

The master TOC calls up the subsidiary TOCs

Note: Template Notes are marked as conditional text (Author Hint) and so are not included in the output.

Targets

There is one Target (PDF). The Target determines:

  • Which TOCs to use

  • Where the output is stored

  • What content to include/exclude

The target dialog box has 12 tabs each with its own set of fields, but you can ignore these. The target is set up so you can just select Build, View or Publish from the main ribbon.

Build

Build compiles the project but does not display the PDF. This is useful to check for errors (see the Build Log on the Builds panel) as it takes time to open and display the PDF output.

View

View compiles the project, creates the PDF and displays it. Note: you must close the PDF viewer before doing another view or build.

Publish

Publish takes the compiled output and sends it to a new location, for example this could update a website or send it to a network location.

Excel spreadsheets

You can import an Excel spreadhseet into Flare. You can choose wheteher the imorted file is linked to the source file or not. If it is linked you can referesh the file which reimports the latest version of the soucr. You can set the refresh to occur automatically each time an output is generated. (Actually, you can do this with most file types).

When you import and link a spreadsheet, you can choose which sheets cells and charts to import and or link. You can even exclude certain cells from the displayed result.

Import a spreadsheet

Use the Project>Import>Ms_Excel>Spreadsheet tool.

Accept the default: Import into this project. Click next.

Click on and browse to the Excel file to be imported.

Check that the "Link generated files to source files box is ticked.

Click Next

 

Check the box Auto-reimport before 'Generate Output'.

Click Next.

Select Don't preserve MS Excel styles.

Click Next.

A preview of the files to be imported is shown. Any charts or graphs are stored in the Content Explorer>Resources>Images folder. Excel files are converted to HTML tables and stored in a new topic file within a new folder such as: Content>2022-09-23 Excel chart source>Sheet1.htm.

Pinned Styles

You can pin various elements such as styles, images, snippets such that they appear at the top of the pick list. Pinned elements are unique to each user and are stored in a folder under Project>Users (Note that this folder is not visible within the Flare Project Organiser).

When a different user opens a project for the first time, the pinned styles will need repinning.

To avoid this, a pinned style for each user can be created in the template project. (Assumes youknow the Windows user name of each user).

Open the template project in Windows File Explorer

Browse to My Project>....>Project>Users

Create a new folder with the name of the Windows users (For example, myu Windows user name is thbri)

Paste in the following file: PinnedStyles.xml

This is a simple text file with typical content of:

<PinnedItems>
<PinnedItem Name="p.BodyHeading" />
