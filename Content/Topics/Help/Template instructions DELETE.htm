Inline formatting (there shouldn't be any!) is shown inside square brackets. To see spaces, click the ¶ symbol on the editor mini-ribbon bar.

Table gridlines – some template elements are based on tables without borders which will show as dotted lines. These will not print. 

Text pasted into a topic will lose any formatting. Assign styles as appropriate.

Title page

The footer statement (Private & Confidential statement) is held as a variable. To remove this statement, click on Project Organizer>Variables> and delete the text.

The Document Title, Sub Title and Date are held as Variables. To change, click on Project Organizer>Variables.

The title page is built in a table structure. Do not delete any of the holding cells and do not delete entire front page.

The document title and subtitle text should not exceed more than two lines of text.

The title page is pre-set with two large logo images at the bottom of the page. Please ensure your final report document always displays only one image and the other one is deleted. This is the same for any Section Dividers that are inserted. When one logo is deleted the other one will right align in the correct position.

Contents page

Position your cursor inside the table of contents and press F9 to update. This will need to be done for the Contents and also for both the Tables and Figures.

The table of contents, table of tables and table of figures are compiled automatically from heading text styles and caption text styles used throughout the document.

You can delete the table of tables and/or table of figures if not required. To reinsert them you will need to copy and paste from a new template.

Summary page

The optional Summary page is held in a topic and is always set to Page 1.

To remove the Summary page, delete the Summary Page entry in the TOC.

Main body

Working with text:

IMPORTANT: To paste text from other documents, position your cursor on an empty paragraph marker of '~BodyText' style and use 'Paste Special - Unformatted Text'. Then highlight the paragraphs to be formatted and apply the relevant text style from the Quick Style Thumbnails. The style thumbnails on the home ribbon show the most useful template text styles.

Alternatively, show all styles on the right panel by selecting the small 'Arrow bottom right' on the 'Styles' group of the Home ribbon or press ALT+CTRL+SHIFT+S.

Working with heading styles

The Style Sheet has been created with four levels of numbered headings which can be accessed from the Home>Style dropdown, from the Style Window or by right clicking on the element and choosing Style Class.

If you do not need a numbered heading use SumHead and SumSubHead Heading styles, this will only show first level of heading in the Table of Contents.

Working with tables:

Tables inserted with the native Word function can be reformatted to template style by applying the appropriate 'Table Style' from the Table Design Ribbon (~Unite Table – Light or Red). These table styles are further divided into standard and wide layouts.

The standard table will sit inside the text column width whereas the wide tables are designed to use the entire width of the page.

The text style inside will default to the text style of the cursor position, usually Body Text. This may lead to large row height settings. To format table text with better spacing, use either '~TableTextLeft' (for text table) or '~TableTextRight' (for data table) text styles should be applied to any table.

Working with Variables

Variables contain a single element (character, word or phrase) which is used in many places and can be updated once to update in many places. Select ‘Insert Variable’. Flare contains several groups of variables (Document data, universal, user defined) Selecting a Variable will insert the item at the cursor position.

Working with Snippets

Snippets are preformed blocks of text. Snippets can contain commonlu used words. sentences, paragraphs, quotes, tables, graphics. Select ‘Insert Snippet’ for a selection of pre-set textual and graphical items. Selecting a Snippet will insert the item at cursor position. Always best to place your cursor on an empty paragraph. Snippets can also contain variables.

Graphical items

Quotes, Key Messages and Action Points

Insert pre-formatted highlight text placeholders by positioning your cursor in empty body text paragraph marker. Then either insert one from the Quick Parts Gallery, type or paste unformatted text at the prompt.

Graphic elements

Graphical elements (including illustrations, photographs and multi-media) are inserted using Insert>Image or Insert>Multimedia or Insert>screen capture. Figure captions are created using the style p.FigureTitle. Numbering of figures is automatic and these will appear in the table of cotnenst under the sub-heading Figures.

The figure caption and source can be omitted if not required.

Flare supports a wide range of image formats, but the preferred is .png for raster graphics (photos) and .edf for vector graphics (diagrams).

Diagrams must be created outside of Flare and imported.

Working with landscape layouts

Two pre-set topics are included for landscape pages (in A4 and A3 paper size). These are held in the Resources>TopicTemplates folder. Copy the topic into your working topics folder.

Follow the instructions included on the landscape page for corrections on the page numbering (if required).

Working with Section Dividers and Running Headings

The template has been designed to repeat the title of the Section Divider in the header of the page. Without any Section Dividers the document title will repeat in the header. If using Section Dividers, you will need to insert one immediately after the optional Summary section to ensure the Headers work properly.

Headers and Footers

Document title and Confidential statement are linked to the topic's page layout.

Explained further below, if the report has section divider pages, the header on the inside pages will automatically reflect the section divider title instead of document title.

Working with appendices

Insert a new Appendix Section by copying the topic template. This will create an Appendix Section which will be Appendix A.

Subsequent Appendix topics will be numbered automatically.

All appendix titles should be formatted with the style 'AppHead' which will automatically apply the alphabetic bullet to each heading.
